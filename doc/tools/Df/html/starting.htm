More Informations

The DF GUI offers access to DF features using the menu bar and the tool bar visible.The gui shows a tabbed pane with three different views of the funciotns provided by a Directory Facilitator. Three views are: 

  • Agents registered with the DF:  shows a table with all the agents registered with the DF. 
  • Last Search Results: shows a table with the list of agent descriptors that were returned as a result of the last search operation.
  • DF Federation: shows the DF federation. The Parents table shows the list of DF's with which this DF is federated, while the Children table shows the list of DF's that are registered with this DF.

The menus available are:

1.General Menu
This menu contains the general commands that are always allowed from any of the three views:

  1. Exit DF

    2. This item terminates the DF agent, invoking its doDelete() method.
  1. Close GUI

    2. This item only distroy the GUI of this DF.
2.Catalogue
3.Super DF
For this two menus according to the view (tab selected) only some actions are allowed.

VIEWS AND ACTION ALLOWED:

1. AGENTS REGISTERED WITH THE DF:

1.Catalogue Menu:
    1. View ( )
              This item permits to view the description of the selected agent from the table. 
    1. Modify ()

      2. This item of the menu allows to modify the description of the selected agent.
    2. Register ()

      3. This item  permits to register an agent with the DF. The user is then requested to fill in an agent description, notice that some values are mandatory for registration.
    3. Deregister ()

      4. This item allows to deregister an agent selected in the table. 
    4. Search ()

      5. This item allows to search for agent descriptions with this DF. If no value is inserted in the agent description, the search action returns all the active agents currently registered with this DF. 
    5. Search with constraints ()

      6. This item of the menu allows to make a search with this DF by adding further constraints, as specified by the FIPA specifications. The constraints inserted are stored to avoid inserting them again for the next operation. Two kinds of constraints are permitted df-depth: the depth of propagation of the search operation to the federated DF's, and the resp-req: the number of returned agent descriptions. 


    2. Super DF:
     

    1. Federate ()

      2. This function allows to federate this DF with another DF. First of all, the user must provide the full name of the DF with which to federate and then the description of this DF that must be registered with the specified DF

2. Last Search Results:

1.Catalogue Menu:
  1. View 

    2. This item allows to view the description of a selected agent on the table of the results. 
  2. Search  (see above) 
  3. Search with Constraints (see above).
2. Super DF:
     No action allowed.

3. DF Federation:

1.Catalogue Menu:
  1. View 

    2. This item allows view the description of an agent selected in one of the two tables. If the agent selected is a parent, then the default description of this DF is shown. Otherwise if the selected agent is a child, then the description of this child DF is shown. 
  2. Deregister

    3. This item allows to make a deregister action. If the selected agent is a parent then this DF is deregistered from the selected one, otherwise, if the agent selected is a child, this child is deregistered from this DF. 
  3. Search 

    4. This item permits to make a search with default constraint with the DF selected in one of the tables. 
  4. Search with constraints 

    5. This item permits to make a search with constraints with the DF selected in one of the tables. 
2. Super DF:
  1. Federate 

    2. This item allows to federate this DF with the selected one.

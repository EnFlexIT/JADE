Mobile

Author: Giovanni Caire (TILab S.p.a)
Date: 08/08/2001

This example shows how to build mobile agents, which are able to migrate or copy themself across multiple network host. In this version of JADE only intra-platform mobility is supported. An agent can navigate acroos different agent containers but is confined to a single JADE platform(see the programmer's guide for more details on Agent Mobility).

To launch this examples, the user can activate the main container (java jade.Boot -platform), several agent container (java jade.Boot -container) and then start MobileAgent from command line.Through the agent's GUI it is possible to select the location (i.e. container) on which move or clone the agent.

 

MobileAgent
Description:
This class contains the two resources used by the agent behaviours: the counter cnt and the flag cntEnabled. At the setup it adds two behaviours to serve the incoming messages and to increment the counter. In particular, notice the usage of the two methods beforeMove() and afterMove() to execute some application-specific tasks just before  and just after the agent migration takes effect.
Jade classes used:
mobilityOntology,Codec,SL0Codec
Support Classes
CounterBehaviour,GetAvailableLocationsBehaviour,ServeIncomingMessagesBehaviour,MobileAgentGui,LocationTableModel

MessageTemplate

    This example shows the user the way to define an application specific MessageTemplate.
    The WaitAgent agent waits for a REQUEST message from a set of agents the user must provide.Since the default method in class MessageTemplate does not provide a way to define such a pattern, the example shows how to create this application specific pattern.
    To try the example, the user can start the WaitAgent (from command line when launching the platform or via RMA gui ) and  then enters the name of the agents that will send messages to this agent (for example start a DummyAgent from the RMA gui and use it as sender).
    If an REQUEST message will be sent (from one of the expected agent) than the WaitAgent will reply with a INFORM message otherwise no reply

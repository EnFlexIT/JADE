MessageTemplate

Author: Tiziana Trucco (TILab S.p.a)
Date: 17/09/2001

This example shows the way to use the MessageTemplate.MatchExpression Interface to define an application specific MessageTemplate. In particular in the method match() the programmer should realize the necessary checks on the ACLMessage in order to return true if the message match with the application specific requirement

To try the example, the user can start the WaitAgent (from command line when launching the platform or via RMA gui ) and  then enters the name of the agents that will send messages to this agent (for example start a DummyAgent from the RMA gui and use it as sender).

 
 

WaitAgent
Description:
The WaitAgent agent waits for a REQUEST message from a set of agents the user must provide.If a REQUEST message will be sent (from one of the expected agent) than the WaitAgent will reply with a INFORM message otherwise no reply. Since the default method in class MessageTemplate does not provide a way to define such a pattern, the example shows how to create this application specific pattern.
Jade classes used:
SimpleBehaviour,MessageTemplate.MatchExpression
Source
\src\examples\MessageTemplate\WaitAgent.java
